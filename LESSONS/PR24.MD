# 🧪 Практическая работа №24  
## Создание зашифрованной резервной копии

⏱️ **Продолжительность:** 90 минут  
🎯 **Цель работы:**  
Научиться создавать зашифрованные резервные копии с использованием сертификатов, управлять ключами шифрования, создавать резервные копии сертификатов, а также восстанавливать зашифрованные бэкапы на том же и на другом сервере.

🖥️ **Оборудование:**  
Виртуальная машина с Windows Server / Windows 10, установленными SQL Server Developer Edition и SSMS.

---

## 🧠 Глубокое погружение: перед началом

Шифрование бэкапов — это не просто "галочка" в настройках, а полноценная система защиты. Если вы потеряете ключ шифрования, вы потеряете данные навсегда. В этой работе вы научитесь:

- Создавать иерархию шифрования (мастер-ключ, сертификат)
- Создавать зашифрованные бэкапы
- Управлять резервными копиями сертификатов
- Восстанавливать данные на том же и другом сервере

---

## 📋 Ход выполнения работы

### 🔹 Этап 1. Подготовка среды (10 мин)

1. Запустите **SSMS** и подключитесь к экземпляру SQL Server.

2. Создайте тестовую базу данных:

```sql
CREATE DATABASE EncryptionPractice_Фамилия;
GO
USE EncryptionPractice_Фамилия;
GO
```

3. Создайте таблицы и наполните их данными:

```sql
CREATE TABLE dbo.SecretData (
    ID INT IDENTITY(1,1) PRIMARY KEY,
    SecretText NVARCHAR(500),
    Created DATETIME DEFAULT GETDATE()
);

INSERT INTO dbo.SecretData (SecretText)
VALUES 
    ('Конфиденциальная информация 1'),
    ('Конфиденциальная информация 2'),
    ('Конфиденциальная информация 3'),
    ('Конфиденциальная информация 4'),
    ('Конфиденциальная информация 5');
GO
```

4. Создайте папку `C:\Backup\ПР24\` для хранения бэкапов:

```sql
EXEC xp_cmdshell 'mkdir C:\Backup\ПР24\';  -- если xp_cmdshell включён
```

5. **Сделайте скриншот** создания базы и таблицы.

### 🔹 Этап 2. Создание иерархии шифрования (15 мин)

2.1. **Создайте мастер-ключ базы данных в master:**

```sql
USE master;
GO

-- Создаём мастер-ключ (если ещё не создан)
IF NOT EXISTS (SELECT * FROM sys.symmetric_keys WHERE name = '##MS_DatabaseMasterKey##')
BEGIN
    CREATE MASTER KEY ENCRYPTION BY PASSWORD = 'MasterKey_ВашаФамилия_2024!';
    PRINT 'Master key created';
END
ELSE
    PRINT 'Master key already exists';
GO
```

2.2. **Проверьте наличие мастер-ключа:**

```sql
SELECT * FROM sys.symmetric_keys WHERE name = '##MS_DatabaseMasterKey##';
```

2.3. **Создайте сертификат для шифрования бэкапов:**

```sql
CREATE CERTIFICATE BackupEncryptionCert
WITH SUBJECT = 'Certificate for backup encryption - ' + @@SERVERNAME,
     EXPIRY_DATE = '2027-12-31';
GO

-- Проверяем создание
SELECT * FROM sys.certificates WHERE name = 'BackupEncryptionCert';
```

2.4. **Создайте симметричный ключ (опционально, для понимания):**

```sql
CREATE SYMMETRIC KEY BackupSymKey
WITH ALGORITHM = AES_256
ENCRYPTION BY CERTIFICATE BackupEncryptionCert;
GO

SELECT * FROM sys.symmetric_keys WHERE name = 'BackupSymKey';
```

2.5. **Сделайте скриншот** результатов создания ключей и сертификата.

---

### 🔹 Этап 3. Создание зашифрованной резервной копии (15 мин)

3.1. **Создайте полную резервную копию с шифрованием:**

```sql
BACKUP DATABASE [EncryptionPractice_Фамилия]
TO DISK = N'C:\Backup\ПР24\EncryptionPractice_Фамилия_Encrypted.bak'
WITH 
    COMPRESSION,
    ENCRYPTION (
        ALGORITHM = AES_256,
        SERVER CERTIFICATE = BackupEncryptionCert
    ),
    STATS = 10;
```

3.2. **Создайте бэкап журнала с шифрованием (для полноты):**

```sql
BACKUP LOG [EncryptionPractice_Фамилия]
TO DISK = N'C:\Backup\ПР24\EncryptionPractice_Фамилия_Log_Encrypted.trn'
WITH 
    COMPRESSION,
    ENCRYPTION (
        ALGORITHM = AES_256,
        SERVER CERTIFICATE = BackupEncryptionCert
    );
```

3.3. **Создайте также незашифрованный бэкап для сравнения:**

```sql
BACKUP DATABASE [EncryptionPractice_Фамилия]
TO DISK = N'C:\Backup\ПР24\EncryptionPractice_Фамилия_Plain.bak'
WITH COMPRESSION;
```

3.4. **Сравните размеры файлов** (через Проводник или xp_cmdshell):

```sql
EXEC xp_cmdshell 'dir C:\Backup\ПР24\*.bak';
```

3.5. **Сделайте скриншоты** выполнения команд и сравнения размеров.

### 🔹 Этап 4. Резервное копирование сертификата (15 мин)

**Важно!** Это самый критический этап. Без резервной копии сертификата вы не сможете восстановить бэкап на другом сервере.

4.1. **Создайте резервную копию сертификата и приватного ключа:**

```sql
BACKUP CERTIFICATE BackupEncryptionCert
TO FILE = N'C:\Backup\ПР24\BackupEncryptionCert.cer'
WITH PRIVATE KEY (
    FILE = N'C:\Backup\ПР24\BackupEncryptionCert_PrivateKey.pvk',
    ENCRYPTION BY PASSWORD = N'CertKey_ВашаФамилия_2024!'
);
```

4.2. **Проверьте, что файлы создались:**

```sql
EXEC xp_cmdshell 'dir C:\Backup\ПР24\*.cer';
EXEC xp_cmdshell 'dir C:\Backup\ПР24\*.pvk';
```

4.3. **Создайте скрипт для восстановления сертификата (сохраните в файл):**

```sql
-- Скрипт для восстановления сертификата на другом сервере
-- Сохраните этот скрипт в папку с бэкапами!

USE master;
GO

-- 1. Создать мастер-ключ (если нет)
IF NOT EXISTS (SELECT * FROM sys.symmetric_keys WHERE name = '##MS_DatabaseMasterKey##')
BEGIN
    CREATE MASTER KEY ENCRYPTION BY PASSWORD = 'MasterKey_ВашаФамилия_2024!';
END
GO

-- 2. Восстановить сертификат
CREATE CERTIFICATE BackupEncryptionCert
FROM FILE = N'C:\Backup\ПР24\BackupEncryptionCert.cer'
WITH PRIVATE KEY (
    FILE = N'C:\Backup\ПР24\BackupEncryptionCert_PrivateKey.pvk',
    DECRYPTION BY PASSWORD = N'CertKey_ВашаФамилия_2024!'
);
GO
```

4.4. **Сделайте скриншот** содержимого папки с файлами сертификата.

### 🔹 Этап 5. Восстановление зашифрованного бэкапа (20 мин)

5.1. **Удалите исходную базу** (имитация восстановления на новый сервер):

```sql
USE master;
GO
DROP DATABASE EncryptionPractice_Фамилия;
GO
```

5.2. **Проверьте, что сертификат существует:**

```sql
SELECT * FROM sys.certificates WHERE name = 'BackupEncryptionCert';
```

5.3. **Попробуйте восстановить базу из зашифрованного бэкапа:**

```sql
RESTORE DATABASE EncryptionPractice_Фамилия_Restored
FROM DISK = N'C:\Backup\ПР24\EncryptionPractice_Фамилия_Encrypted.bak'
WITH 
    MOVE 'EncryptionPractice_Фамилия' TO 'C:\Program Files\Microsoft SQL Server\MSSQL15.MSSQLSERVER\MSSQL\DATA\EncryptionPractice_Фамилия_Restored.mdf',
    MOVE 'EncryptionPractice_Фамилия_log' TO 'C:\Program Files\Microsoft SQL Server\MSSQL15.MSSQLSERVER\MSSQL\DATA\EncryptionPractice_Фамилия_Restored_log.ldf',
    RECOVERY;
```

5.4. **Проверьте восстановленные данные:**

```sql
USE EncryptionPractice_Фамилия_Restored;
GO
SELECT * FROM dbo.SecretData;
```

5.5. **Сделайте скриншот** успешного восстановления.

### 🔹 Этап 6. Имитация восстановления на другом сервере (15 мин)

В этом этапе мы смоделируем ситуацию, когда нужно восстановить базу на другой сервер. Для этого:

1. Создайте **новую базу данных** (имитация другого сервера)
2. Удалите сертификат (имитация отсутствия на новом сервере)
3. Восстановите сертификат из резервной копии
4. Восстановите базу

6.1. **Удалите сертификат (имитация нового сервера):**

```sql
-- Это опасная команда! Только в учебных целях!
DROP CERTIFICATE BackupEncryptionCert;
GO

-- Проверяем, что сертификата нет
SELECT * FROM sys.certificates WHERE name = 'BackupEncryptionCert';
```

6.2. **Попробуйте восстановить базу (должна быть ошибка):**

```sql
RESTORE DATABASE EncryptionPractice_Фамилия_NewServer
FROM DISK = N'C:\Backup\ПР24\EncryptionPractice_Фамилия_Encrypted.bak'
WITH 
    MOVE 'EncryptionPractice_Фамилия' TO 'C:\Program Files\Microsoft SQL Server\MSSQL15.MSSQLSERVER\MSSQL\DATA\EncryptionPractice_Фамилия_NewServer.mdf',
    MOVE 'EncryptionPractice_Фамилия_log' TO 'C:\Program Files\Microsoft SQL Server\MSSQL15.MSSQLSERVER\MSSQL\DATA\EncryptionPractice_Фамилия_NewServer_log.ldf',
    RECOVERY;
```

**Запишите текст ошибки.** Ошибка будет связана с отсутствием сертификата.

6.3. **Восстановите сертификат из резервной копии:**

```sql
CREATE CERTIFICATE BackupEncryptionCert
FROM FILE = N'C:\Backup\ПР24\BackupEncryptionCert.cer'
WITH PRIVATE KEY (
    FILE = N'C:\Backup\ПР24\BackupEncryptionCert_PrivateKey.pvk',
    DECRYPTION BY PASSWORD = N'CertKey_ВашаФамилия_2024!'
);
```

6.4. **Повторите восстановление** — теперь оно должно пройти успешно.

6.5. **Проверьте данные:**

```sql
USE EncryptionPractice_Фамилия_NewServer;
GO
SELECT * FROM dbo.SecretData;
```

6.6. **Сделайте скриншоты** ошибки, восстановления сертификата и успешного восстановления.

## 📌 Важные замечания

### Безопасность ключей:
- Никогда не храните файлы ключей (.pvk) вместе с бэкапами
- Используйте сложные пароли (не менее 12 символов, цифры, спецсимволы)
- В реальной работе храните ключи в HSM (Hardware Security Module) или защищённом хранилище

### Если xp_cmdshell отключён:
- Создавайте папки и проверяйте файлы вручную через Проводник
- Команды с xp_cmdshell можно пропустить, заменив визуальной проверкой

### Для проверки размера файлов:
- Откройте Проводник → C:\Backup\ПР24\
- Посмотрите свойства файлов
- Сделайте скриншот

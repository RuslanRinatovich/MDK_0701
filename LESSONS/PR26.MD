# 🧪 Практическая работа №26  
## Запуск и проверка работы SQL Server Agent

⏱️ **Продолжительность:** 90 минут  
🎯 **Цель работы:**  
Научиться проверять статус SQL Server Agent, управлять его запуском и остановкой, просматривать журналы ошибок, а также знакомиться с основными элементами управления в SSMS.

🖥️ **Оборудование:**  
Виртуальная машина с Windows Server / Windows 10, установленными SQL Server Developer Edition и SSMS.

## 🧠 Глубокое погружение: перед началом

SQL Server Agent — ключевой компонент для автоматизации. В этой работе вы научитесь управлять этой службой, диагностировать проблемы и просматривать логи. Все действия безопасны и могут выполняться на учебном сервере.

---

## 📋 Ход выполнения работы

### 🔹 Этап 1. Проверка статуса SQL Server Agent (15 мин)

1. Запустите **SSMS** и подключитесь к экземпляру SQL Server.

2. **Визуальная проверка:**  
   В обозревателе объектов найдите узел **SQL Server Agent**. Если рядом зелёный треугольник → Agent запущен. Если красный квадрат → остановлен.

3. **Проверка через T-SQL:**

```sql
-- Проверка статуса Agent
EXEC xp_servicecontrol 'QUERYSTATE', 'SQLSERVERAGENT';
```

4. **Проверка через системное представление:**

```sql
SELECT servicename, service_account, status_desc
FROM sys.dm_server_services
WHERE servicename LIKE '%SQLSERVERAGENT%';
```

5. **Сделайте скриншот** результатов выполнения команд.

---

### 🔹 Этап 2. Управление службой (15 мин)

2.1. **Остановите SQL Server Agent** (только учебная среда!):

```sql
EXEC xp_servicecontrol 'STOP', 'SQLSERVERAGENT';
```

2.2. **Подождите 5 секунд и проверьте статус:**

```sql
EXEC xp_servicecontrol 'QUERYSTATE', 'SQLSERVERAGENT';
```

2.3. **Запустите Agent обратно:**

```sql
EXEC xp_servicecontrol 'START', 'SQLSERVERAGENT';
```

2.4. **Проверьте, что Agent снова запущен.**

2.5. **Альтернативный способ через SSMS:**  
   ПКМ на `SQL Server Agent` → Start / Stop.

2.6. **Сделайте скриншоты** остановленного и запущенного состояния.

⚠️ **Важно:** Не останавливайте Agent, если на сервере запланированы важные задания!

---

### 🔹 Этап 3. Просмотр настроек и свойств Agent (15 мин)

3.1. **Откройте свойства Agent:**  
   ПКМ на `SQL Server Agent` → Properties.

3.2. **Изучите вкладки:**
   - **General:** Настройки авто-перезапуска
   - **Advanced:** Уровень логирования, файл лога
   - **Alert System:** Для уведомлений
   - **Connection:** Параметры подключения

3.3. **Запишите** путь к файлу журнала ошибок (`SQLAGENT.OUT`) со вкладки Advanced.

3.4. **Просмотрите настройки через T-SQL:**

```sql
-- Вывод свойств Agent
EXEC msdb.dbo.sp_help_sqlagent_properties;
```

3.5. **Сделайте скриншот** окна свойств.

---

### 🔹 Этап 4. Анализ журналов SQL Agent (20 мин)

4.1. **Физический журнал ошибок:**  
   Откройте файл `SQLAGENT.OUT` в Блокноте (путь из предыдущего шага). Найдите последние записи.

4.2. **Просмотр через T-SQL:**

```sql
-- Чтение лога Agent (0-текущий файл, 2-агент)
EXEC xp_readerrorlog 0, 2, 'Error';
```

4.3. **Создание событий в логе (для теста):**

```sql
-- Вызовет запись в логе Agent
RAISERROR('Тестовое событие в лог SQL Agent', 16, 1) WITH LOG;
```

4.4. **Проверьте, появилась ли запись в журнале.**

4.5. **Просмотр ошибок запуска заданий:**

```sql
SELECT TOP 10 *
FROM msdb.dbo.sysjobhistory
WHERE run_status = 0  -- ошибка
ORDER BY run_date DESC, run_time DESC;
```

4.6. **Сделайте скриншот** содержимого журнала.

---

### 🔹 Этап 5. Изучение компонентов Agent в msdb (15 мин)

5.1. **Переключитесь на базу msdb:**

```sql
USE msdb;
GO
```

5.2. **Просмотрите таблицы, связанные с Agent:**

```sql
-- Количество заданий
SELECT COUNT(*) AS TotalJobs FROM msdb.dbo.sysjobs;

-- Количество расписаний
SELECT COUNT(*) AS TotalSchedules FROM msdb.dbo.sysschedules;

-- Количество операторов
SELECT COUNT(*) AS TotalOperators FROM msdb.dbo.sysoperators;

-- Количество оповещений
SELECT COUNT(*) AS TotalAlerts FROM msdb.dbo.sysalerts;
```

5.3. **Просмотрите свойства журнала заданий:**

```sql
-- Максимальное количество строк в истории
EXEC msdb.dbo.sp_help_sqlagent_properties;
-- Обратите внимание на job_history_max_rows
```

5.4. **Создайте простое задание для теста (без расписания):**

```sql
USE msdb;
GO
EXEC dbo.sp_add_job
    @job_name = N'TestAgent_Фамилия',
    @description = N'Тестовое задание для проверки работы Agent',
    @owner_login_name = N'sa';

EXEC dbo.sp_add_jobstep
    @job_name = N'TestAgent_Фамилия',
    @step_name = N'Test Step',
    @command = N'SELECT GETDATE();';

EXEC dbo.sp_add_jobserver
    @job_name = N'TestAgent_Фамилия';
```

5.5. **Запустите задание вручную:**

```sql
EXEC msdb.dbo.sp_start_job @job_name = N'TestAgent_Фамилия';
```

5.6. **Проверьте историю:**

```sql
SELECT 
    j.name,
    jh.run_date,
    jh.run_status,
    jh.message
FROM msdb.dbo.sysjobs j
JOIN msdb.dbo.sysjobhistory jh ON j.job_id = jh.job_id
WHERE j.name = 'TestAgent_Фамилия'
ORDER BY jh.run_date DESC, jh.run_time DESC;
```

5.7. **Удалите тестовое задание:**

```sql
EXEC msdb.dbo.sp_delete_job @job_name = N'TestAgent_Фамилия';
```

5.8. **Сделайте скриншот** истории выполнения тестового задания.

## 💡 Полезные советы

### Если Agent не запускается:
- Проверьте, запущен ли SQL Server
- Проверьте учётную запись службы
- Посмотрите журнал событий Windows: `eventvwr.msc`

### Если xp_servicecontrol не работает:
```sql
-- Включить xp_cmdshell
EXEC sp_configure 'xp_cmdshell', 1; RECONFIGURE;
-- Тогда можно так
EXEC xp_cmdshell 'net start SQLSERVERAGENT';
```

### Просмотр логов SQL Server и Agent:
- В SSMS: Management → SQL Server Logs
- Или через `xp_readerrorlog`

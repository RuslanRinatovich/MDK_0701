# 🧪 Практическая работа №20  
## Проверка резервной копии; восстановление на тестовую БД

⏱️ **Продолжительность:** 90 минут  
🎯 **Цель работы:**  
Научиться проверять целостность резервных копий с помощью `RESTORE VERIFYONLY`, выполнять диагностику баз данных командой `DBCC CHECKDB`, а также восстанавливать резервные копии на тестовую среду для проверки их работоспособности. Освоить методы проактивного контроля целостности данных и резервных копий.

🖥️ **Оборудование:**  
Виртуальная машина с Windows Server / Windows 10, установленными SQL Server Developer Edition и SSMS.

- Проверять целостность баз данных (`DBCC CHECKDB`)

- Моделировать повреждения и диагностировать их
- Создавать проактивную систему мониторинга целостности

```sql
-- Представление для статистики по отделам
CREATE VIEW dbo.vwDepartmentStats AS
SELECT 
    Department,
    COUNT(*) AS EmployeeCount,
    AVG(Salary) AS AvgSalary,
    MIN(Salary) AS MinSalary,
    MAX(Salary) AS MaxSalary
FROM dbo.Employees
GROUP BY Department;
GO

-- Представление для загрузки проектов
CREATE VIEW dbo.vwProjectLoad AS
SELECT 
    p.ProjectID,
    p.ProjectName,
    p.Budget,
    COUNT(a.AssignmentID) AS EmployeeCount,
    SUM(CASE WHEN a.Role = 'Руководитель' THEN 1 ELSE 0 END) AS ManagersCount
FROM dbo.Projects p
LEFT JOIN dbo.Assignments a ON p.ProjectID = a.ProjectID
GROUP BY p.ProjectID, p.ProjectName, p.Budget;
GO

-- Процедура для повышения зарплаты в отделе
CREATE PROCEDURE dbo.RaiseSalaryByDepartment
    @Department NVARCHAR(50),
    @Percent INT
AS
BEGIN
    UPDATE dbo.Employees
    SET Salary = Salary * (1 + @Percent / 100.0)
    WHERE Department = @Department;
    
    SELECT @@ROWCOUNT AS UpdatedCount;
END;
GO

-- Процедура для отчёта по проектам
CREATE PROCEDURE dbo.GetProjectReport
    @MinBudget DECIMAL(15,2) = 0
AS
BEGIN
    SELECT 
        p.ProjectName,
        p.Budget,
        COUNT(DISTINCT a.EmployeeID) AS EmployeeCount,
        STRING_AGG(DISTINCT e.Department, ', ') AS Departments
    FROM dbo.Projects p
    LEFT JOIN dbo.Assignments a ON p.ProjectID = a.ProjectID
    LEFT JOIN dbo.Employees e ON a.EmployeeID = e.EmployeeID
    WHERE p.Budget >= @MinBudget
    GROUP BY p.ProjectName, p.Budget
    ORDER BY p.Budget DESC;
END;
GO
```

```powershell

$filePath = "C:\Program Files\Microsoft SQL Server\MSSQL15.MSSQLSERVER\MSSQL\DATA\IntegrityCheck_Фамилия.mdf"

# Открываем файл для записи

# Перемещаемся на смещение 1 МБ (имитация повреждения)

# Записываем мусор (100 байт случайных данных)

(New-Object Random).NextBytes($buffer)
```

### 🔹 Этап 6. Комплексный скрипт для автоматической проверки (10 мин)

**Задание:** Создайте скрипт, который автоматически проверяет все базы и все резервные копии.

6.1. **Проверка всех пользовательских баз на сервере:**

```sql
-- Создадим временную таблицу для результатов
CREATE TABLE #DBCCResults (
    DatabaseName NVARCHAR(128),
    CheckTime DATETIME,
    Status NVARCHAR(50),
    ErrorMessage NVARCHAR(MAX)
);

-- Курсор по всем пользовательским базам
DECLARE @dbName NVARCHAR(128);
DECLARE db_cursor CURSOR FOR
    SELECT name 
    FROM sys.databases 
    WHERE database_id > 4 
        AND state_desc = 'ONLINE'
        AND name LIKE '%Фамилия%';  -- фильтр для ваших БД

OPEN db_cursor;
FETCH NEXT FROM db_cursor INTO @dbName;

WHILE @@FETCH_STATUS = 0
BEGIN
    BEGIN TRY
        INSERT INTO #DBCCResults (DatabaseName, CheckTime, Status)
        VALUES (@dbName, GETDATE(), 'CHECKING');
        
        -- Выполняем быструю проверку
        EXEC ('DBCC CHECKDB (' + @dbName + ') WITH PHYSICAL_ONLY, NO_INFOMSGS');
        
        UPDATE #DBCCResults 
        SET Status = 'OK' 
        WHERE DatabaseName = @dbName AND CheckTime = (SELECT MAX(CheckTime) FROM #DBCCResults WHERE DatabaseName = @dbName);
    END TRY
    BEGIN CATCH
        UPDATE #DBCCResults 
        SET Status = 'ERROR', 
            ErrorMessage = ERROR_MESSAGE()
        WHERE DatabaseName = @dbName AND CheckTime = (SELECT MAX(CheckTime) FROM #DBCCResults WHERE DatabaseName = @dbName);
    END CATCH
    
    FETCH NEXT FROM db_cursor INTO @dbName;
END

CLOSE db_cursor;
DEALLOCATE db_cursor;

-- Результаты
SELECT * FROM #DBCCResults ORDER BY CheckTime DESC;
DROP TABLE #DBCCResults;
```

1. Титульный лист
2. Цель работы
3. Ход выполнения (этапы 1–6) с **скриншотами**:
   - Этап 1: стартовый скрипт и создание бэкапов
   - Этап 2: все варианты DBCC CHECKDB (5 скриншотов)
   - Этап 3: все варианты RESTORE VERIFYONLY (5 скриншотов)
   - Этап 4: моделирование повреждения и диагностика (ошибки, suspect_pages)
   - Этап 5: восстановление на тестовую БД и проверка
   - Этап 6: комплексные скрипты проверки
4. Таблица наблюдений
5. Ответы на вопросы
6. Вывод

**Требования к скриншотам:**
- Виден текст команды и результат
- В заголовке окна – имя базы
- Подпись под каждым скриншотом

## 📊 Краткие критерии оценивания

| Критерий | «5» (отлично) | «4» (хорошо) | «3» (удовл.) |
|----------|----------------|---------------|---------------|
| **Стартовый скрипт (этап 1)** | Все объекты созданы, 2 бэкапа, скриншоты | Часть объектов пропущена | Не выполнен |
| **DBCC CHECKDB (этап 2)** | Все 5 запросов, скриншоты | 3-4 запроса | Менее 3 |
| **RESTORE VERIFYONLY (этап 3)** | Все 5 запросов, скриншоты | 3-4 запроса | Менее 3 |
| **Моделирование повреждения (этап 4)** | Все шаги, ошибка 824, suspect_pages | Частично | Не выполнено |
| **Тестовое восстановление (этап 5)** | Успешно, проверка объектов | Восстановлено, но не проверено | Не выполнено |
| **Комплексные скрипты (этап 6)** | Оба скрипта работают | Один скрипт | Не выполнено |
| **Таблица наблюдений** | Полностью заполнена | 1-2 пропуска | Отсутствует |
| **Ответы на вопросы** | Полные, с терминами | 3-4 неполных | Большинства нет |
| **Оформление отчёта** | Структура, подписи, вывод | Мелкие нарушения | Неструктурирован |

**Штрафы:** -1 за отсутствие скриншотов, -0.5 за нечитаемые, -1 за отсутствие вывода

## 📌 Важные замечания по выполнению

### ⚠️ Для этапа 4 (моделирование повреждения):
- Выполняйте только в учебной среде, никогда на production!
- Убедитесь, что у вас есть полная резервная копия перед началом
- Если PowerShell недоступен, можно пропустить этот этап или использовать альтернативный метод

### 💡 Альтернативный метод моделирования повреждения:
Если PowerShell не работает, можно просто остановить базу и вручную подредактировать файл в шестнадцатеричном редакторе (например, HxD), но это сложнее.

### ✅ Для успешного выполнения:
- Всегда проверяйте состояние базы перед операциями
- Делайте скриншоты каждого шага
- Записывайте все ошибки и необычные наблюдения

### 🔹 Этап 3. Создание скрипта для одной базы (15 мин)

**Задание:** Напишите универсальный скрипт для бэкапа одной базы.

3.1. **Создайте скрипт с параметрами:**

```sql
-- Объявляем параметры
DECLARE @DatabaseName NVARCHAR(128) = 'AutoBackup1_Фамилия';
DECLARE @BackupPath NVARCHAR(255) = 'C:\Backup\ПР21\';
DECLARE @CompressionEnabled BIT = 1;
DECLARE @ChecksumEnabled BIT = 1;

-- Генерируем имя файла
DECLARE @DateStr NVARCHAR(20) = FORMAT(GETDATE(), 'yyyyMMdd_HHmm');
DECLARE @FileName NVARCHAR(500) = @BackupPath + @DatabaseName + '_' + @DateStr + '.bak';

-- Формируем команду бэкапа
DECLARE @SQL NVARCHAR(MAX);
SET @SQL = 'BACKUP DATABASE [' + @DatabaseName + '] TO DISK = ''' + @FileName + '''';

IF @CompressionEnabled = 1
    SET @SQL = @SQL + ' WITH COMPRESSION';
ELSE
    SET @SQL = @SQL + ' WITH NO_COMPRESSION';

IF @ChecksumEnabled = 1
    SET @SQL = @SQL + ', CHECKSUM';

SET @SQL = @SQL + ', STATS = 10;';

-- Выводим команду для проверки
PRINT 'Команда бэкапа:';
PRINT @SQL;

-- Выполняем
EXEC sp_executesql @SQL;

-- Выводим информацию о результате
PRINT 'Бэкап завершён: ' + @FileName;
```

### 🔹 Этап 4. Создание хранимой процедуры для бэкапа (15 мин)

**Задание:** Оформите скрипт в виде хранимой процедуры для многократного использования.

4.1. **Создайте процедуру:**

```sql
USE master;
GO

CREATE OR ALTER PROCEDURE dbo.usp_BackupDatabase
    @DatabaseName NVARCHAR(128),
    @BackupPath NVARCHAR(255) = 'C:\Backup\ПР21\',
    @CompressionEnabled BIT = 1,
    @ChecksumEnabled BIT = 1
AS
BEGIN
    SET NOCOUNT ON;
    
    DECLARE @DateStr NVARCHAR(20) = FORMAT(GETDATE(), 'yyyyMMdd_HHmm');
    DECLARE @FileName NVARCHAR(500);
    DECLARE @SQL NVARCHAR(MAX);
    DECLARE @StartTime DATETIME = GETDATE();
    
    -- Проверяем, существует ли база
    IF NOT EXISTS (SELECT 1 FROM sys.databases WHERE name = @DatabaseName)
    BEGIN
        RAISERROR('База данных %s не существует', 16, 1, @DatabaseName);
        RETURN;
    END
    
    -- Генерируем имя файла
    SET @FileName = @BackupPath + @DatabaseName + '_' + @DateStr + '.bak';
    
    -- Формируем команду
    SET @SQL = 'BACKUP DATABASE [' + @DatabaseName + '] TO DISK = ''' + @FileName + '''';
    
    IF @CompressionEnabled = 1
        SET @SQL = @SQL + ' WITH COMPRESSION';
    ELSE
        SET @SQL = @SQL + ' WITH NO_COMPRESSION';
    
    IF @ChecksumEnabled = 1
        SET @SQL = @SQL + ', CHECKSUM';
    
    SET @SQL = @SQL + ', STATS = 10;';
    
    -- Логируем начало
    PRINT 'Начало бэкапа: ' + @DatabaseName + ' в ' + CONVERT(NVARCHAR, @StartTime, 120);
    PRINT 'Файл: ' + @FileName;
    
    -- Выполняем
    BEGIN TRY
        EXEC sp_executesql @SQL;
        
        PRINT 'Бэкап успешно завершён. Длительность: ' + 
              CAST(DATEDIFF(second, @StartTime, GETDATE()) AS NVARCHAR) + ' сек.';
    END TRY
    BEGIN CATCH
        PRINT 'ОШИБКА при бэкапе: ' + ERROR_MESSAGE();
        THROW;
    END CATCH
END;
GO
```

4.2. **Протестируйте процедуру для разных баз:**


### 🔹 Этап 5. Универсальный скрипт для всех баз (20 мин)

**Задание:** Создайте скрипт, который бэкапит все пользовательские базы автоматически.

5.1. **Создайте процедуру для массового бэкапа:**

```sql
CREATE OR ALTER PROCEDURE dbo.usp_BackupAllUserDatabases
    @BackupPath NVARCHAR(255) = 'C:\Backup\ПР21\',
    @CompressionEnabled BIT = 1,
    @ChecksumEnabled BIT = 1,
    @ExcludeDatabases NVARCHAR(MAX) = 'tempdb'
AS
BEGIN
    SET NOCOUNT ON;
    
    DECLARE @DatabaseName NVARCHAR(128);
    DECLARE @SQL NVARCHAR(MAX);
    DECLARE @StartTime DATETIME;
    DECLARE @ErrorMessage NVARCHAR(MAX);
    
    -- Таблица для результатов
    CREATE TABLE #BackupResults (
        DatabaseName NVARCHAR(128),
        StartTime DATETIME,
        EndTime DATETIME,
        Status NVARCHAR(50),
        FileName NVARCHAR(500),
        ErrorMessage NVARCHAR(MAX)
    );
    
    -- Курсор по всем пользовательским БД
    DECLARE db_cursor CURSOR FOR
        SELECT name 
        FROM sys.databases 
        WHERE database_id > 4  -- исключаем системные
            AND state_desc = 'ONLINE'
            AND name NOT IN (SELECT value FROM STRING_SPLIT(@ExcludeDatabases, ','));
    
    OPEN db_cursor;
    FETCH NEXT FROM db_cursor INTO @DatabaseName;
    
    WHILE @@FETCH_STATUS = 0
    BEGIN
        SET @StartTime = GETDATE();
        
        BEGIN TRY
            -- Вызываем процедуру для одной базы
            EXEC dbo.usp_BackupDatabase 
                @DatabaseName = @DatabaseName,
                @BackupPath = @BackupPath,
                @CompressionEnabled = @CompressionEnabled,
                @ChecksumEnabled = @ChecksumEnabled;
            
            -- Логируем успех
            INSERT INTO #BackupResults (DatabaseName, StartTime, EndTime, Status)
            VALUES (@DatabaseName, @StartTime, GETDATE(), 'SUCCESS');
        END TRY
        BEGIN CATCH
            SET @ErrorMessage = ERROR_MESSAGE();
            
            -- Логируем ошибку
            INSERT INTO #BackupResults 
                (DatabaseName, StartTime, EndTime, Status, ErrorMessage)
            VALUES 
                (@DatabaseName, @StartTime, GETDATE(), 'FAILED', @ErrorMessage);
            
            PRINT 'Ошибка при бэкапе ' + @DatabaseName + ': ' + @ErrorMessage;
        END CATCH
        
        FETCH NEXT FROM db_cursor INTO @DatabaseName;
    END
    
    CLOSE db_cursor;
    DEALLOCATE db_cursor;
    
    -- Выводим результаты
    SELECT 
        DatabaseName,
        StartTime,
        EndTime,
        DATEDIFF(second, StartTime, EndTime) AS DurationSeconds,
        Status,
        ErrorMessage
    FROM #BackupResults
    ORDER BY StartTime;
    
    DROP TABLE #BackupResults;
END;
GO
```

5.2. **Выполните массовый бэкап:**

```sql

    @BackupPath = 'C:\Backup\ПР21\',
    @CompressionEnabled = 1,
    @ChecksumEnabled = 1;
```

5.3. **Проверьте, что все файлы создались:**


### 🔹 Этап 6. Создание задания в SQL Server Agent (15 мин)

**Задание:** Настройте автоматическое выполнение бэкапа по расписанию.

6.1. **Проверьте, запущен ли SQL Server Agent:**

```sql
-- В SSMS: в обозревателе объектов должен быть зелёный треугольник у SQL Server Agent
-- Или выполните:
EXEC xp_servicecontrol 'QUERYSTATE', 'SQLSERVERAGENT';
```

6.2. **Создайте задание через T-SQL:**

```sql
USE msdb;
GO

-- Создаём задание
EXEC dbo.sp_add_job
    @job_name = N'AutoBackup_AllDatabases_' + @@SERVERNAME,
    @description = N'Автоматический бэкап всех пользовательских баз',
    @owner_login_name = N'sa',
    @enabled = 1;

-- Создаём шаг задания
EXEC dbo.sp_add_jobstep
    @job_name = N'AutoBackup_AllDatabases_' + @@SERVERNAME,
    @step_name = N'BackupAllDatabases',
    @command = N'
        -- Выполняем наш универсальный скрипт
        EXEC master.dbo.usp_BackupAllUserDatabases 
            @BackupPath = ''C:\Backup\ПР21\'',
            @CompressionEnabled = 1,
            @ChecksumEnabled = 1;
    ',
    @database_name = N'master',
    @on_success_action = 1;  -- перейти к следующему шагу

-- Создаём расписание (каждый день в 22:00)
EXEC dbo.sp_add_schedule
    @schedule_name = N'Nightly_22_00',
    @freq_type = 4,  -- ежедневно
    @freq_interval = 1,
    @active_start_time = 220000;  -- 22:00

-- Привязываем расписание к заданию
EXEC dbo.sp_attach_schedule
    @job_name = N'AutoBackup_AllDatabases_' + @@SERVERNAME,
    @schedule_name = N'Nightly_22_00';

-- Добавляем задание на сервер
EXEC dbo.sp_add_jobserver
    @job_name = N'AutoBackup_AllDatabases_' + @@SERVERNAME;
```

## 💡 Полезные советы

### Если xp_cmdshell не работает:
```sql
-- Включить xp_cmdshell (нужны права sysadmin)
EXEC sp_configure 'show advanced options', 1;
RECONFIGURE;
EXEC sp_configure 'xp_cmdshell', 1;
RECONFIGURE;
```

### Если SQL Agent не запущен:
- В SSMS: правой кнопкой на SQL Server Agent → Start
- Или через Configuration Manager

### Для отладки заданий:
- Запустите задание вручную через `sp_start_job`
- Проверьте историю в `sysjobhistory`
- Посмотрите файловую систему — создались ли файлы

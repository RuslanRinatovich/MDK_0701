# 🧪 Практическая работа №22  
## Настройка резервного копирования в сетевую папку (на одной машине)

⏱️ **Продолжительность:** 90 минут  
🎯 **Цель работы:**  
Научиться настраивать резервное копирование в сетевые папки, работать с UNC-путями, решать проблемы с правами доступа, использовать xp_cmdshell для автоматизации, а также организовывать централизованное хранение бэкапов — всё это на одной виртуальной машине.

🖥️ **Оборудование:**  
Виртуальная машина с Windows Server / Windows 10, установленными SQL Server Developer Edition и SSMS.

---

## 📋 Общая информация

| Параметр | Значение |
|----------|----------|
| Время выполнения | 90 минут |

---

## 🧠 Глубокое погружение: перед началом

Даже на одной машине можно эмулировать сетевое хранение! Windows позволяет обращаться к локальным папкам по сети через специальные адреса (UNC-пути). Это отличный способ научиться работать с правами доступа, не имея второго компьютера.

В этой работе вы научитесь:
- Включать и использовать xp_cmdshell для работы с файловой системой
- Создавать сетевые ресурсы (шары) на локальной машине
- Настраивать права доступа для SQL Server
- Выполнять бэкапы по UNC-путям (`\\localhost\share\`)
- Диагностировать и решать проблемы с доступом

---

## ⚠️ Важное замечание о безопасности

В этой работе мы будем использовать расширенную хранимую процедуру `xp_cmdshell`, которая позволяет выполнять команды операционной системы из SQL Server. По умолчанию она отключена из соображений безопасности. В учебных целях мы научимся её включать, использовать, а затем отключать.

В реальной production-среде:
- Включайте xp_cmdshell только на время выполнения задач
- Сразу отключайте после завершения
- Используйте ограниченные учётные записи
- Согласовывайте действия с администратором безопасности

## 📋 Ход выполнения работы

### 🔹 Этап 1. Подготовка среды (15 мин)

1. Запустите **SSMS** и подключитесь к экземпляру SQL Server.

2. Создайте тестовую базу данных:

```sql
CREATE DATABASE NetworkBackup_Фамилия;
GO
USE NetworkBackup_Фамилия;
GO
```

3. Создайте таблицы и наполните их данными:

```sql
CREATE TABLE dbo.Products (
    ProductID INT IDENTITY(1,1) PRIMARY KEY,
    ProductName NVARCHAR(100),
    Price DECIMAL(10,2),
    Quantity INT
);

CREATE TABLE dbo.Orders (
    OrderID INT IDENTITY(1,1) PRIMARY KEY,
    ProductID INT REFERENCES dbo.Products(ProductID),
    Quantity INT,
    OrderDate DATETIME DEFAULT GETDATE()
);

INSERT INTO dbo.Products (ProductName, Price, Quantity)
VALUES 
    ('Ноутбук', 75000, 10),
    ('Мышь', 1500, 50),
    ('Клавиатура', 3500, 30);

INSERT INTO dbo.Orders (ProductID, Quantity)
VALUES (1, 2), (2, 5), (3, 3);
GO
```

4. **Создание папок через T-SQL с использованием xp_cmdshell**

   Для работы с файловой системой из SQL Server используется расширенная хранимая процедура `xp_cmdshell`. По умолчанию она отключена. Давайте научимся её включать и использовать.

   **4.1. Проверьте текущее состояние xp_cmdshell:**

   ```sql
   -- Проверяем, включён ли xp_cmdshell
   EXEC sp_configure 'xp_cmdshell';
   GO
   ```

   **4.2. Включите xp_cmdshell (требуются права sysadmin):**

   ```sql
   -- Включаем возможность изменения расширенных параметров
   EXEC sp_configure 'show advanced options', 1;
   RECONFIGURE;
   GO
   
   -- Включаем xp_cmdshell
   EXEC sp_configure 'xp_cmdshell', 1;
   RECONFIGURE;
   GO
   
   -- Проверяем, что включилось (значение run_value должно быть 1)
   EXEC sp_configure 'xp_cmdshell';
   GO
   ```

   **4.3. Создайте папки с помощью xp_cmdshell:**

   ```sql
   -- Создаём папку C:\Backup\ПР22\
   EXEC xp_cmdshell 'mkdir C:\Backup\ПР22\';
   GO
   
   -- Создаём подпапку Network
   EXEC xp_cmdshell 'mkdir C:\Backup\ПР22\Network';
   GO
   
   -- Создаём подпапку Local
   EXEC xp_cmdshell 'mkdir C:\Backup\ПР22\Local';
   GO
   ```

   **4.4. Возможные ошибки и их решение:**

   > ⚠️ **Важно!** При выполнении команд могут возникнуть следующие ошибки:

   | Ошибка | Причина | Решение |
   |--------|---------|---------|
   | `SQL Server заблокировал доступ к процедуре "sys.xp_cmdshell"` | xp_cmdshell отключён | Выполните шаг 4.2 |
   | `Access is denied` | Недостаточно прав у учётной записи SQL Server | Запустите SSMS от имени администратора |
   | `The system cannot find the path specified` | Неверный путь | Проверьте, что диск C: существует |

   **4.5. Если папки уже существуют — это не страшно:**

   ```sql
   -- Команда mkdir не вызывает ошибку, если папка уже существует
   -- Просто появится сообщение "Подкаталог или файл уже существует"
   EXEC xp_cmdshell 'mkdir C:\Backup\ПР22\';
   -- Это нормально, работа продолжается
   ```

   **4.6. Проверьте, что папки создались:**

   ```sql
   -- Посмотрим содержимое папки C:\Backup\
   EXEC xp_cmdshell 'dir C:\Backup\';
   GO
   ```

5. **Альтернативный способ: создание папок через Проводник**

   Если по каким-то причинам не удаётся включить xp_cmdshell (например, нет прав sysadmin), можно создать папки вручную:

   - Откройте **Проводник** (Win + E)
   - Перейдите на диск `C:\`
   - Создайте папку `Backup` (правой кнопкой → Создать → Папку)
   - Войдите в `C:\Backup` и создайте папку `ПР22`
   - В папке `C:\Backup\ПР22` создайте папки `Network` и `Local`

6. **Сделайте скриншот** результатов выполнения команд или Проводника с созданными папками.

### 🔹 Этап 2. Создание и настройка локальной сетевой папки (15 мин)

**Создание сетевого ресурса (шары) через Проводник:**

2.1. **Откройте Проводник** и найдите папку `C:\Backup\ПР22\Network`

2.2. **Настройте общий доступ к папке:**
   - Правой кнопкой мыши на папке `Network` → **Свойства**
   - Перейдите на вкладку **Доступ** (Sharing)
   - Нажмите кнопку **Общий доступ...** (Share...)
   - В выпадающем списке выберите **Everyone** (Все)
   - Нажмите кнопку **Добавить** (Add)
   - В столбце "Уровень разрешений" измените на **Чтение и запись** (Read/Write)
   - Нажмите кнопку **Поделиться** (Share)
   - В открывшемся окне появится сетевое имя (например: `\\DESKTOP-ABC123\Network` или `\\localhost\Network`)
   - Нажмите **Готово**

2.3. **Альтернативный способ** (если не получается через мастер):
   - В свойствах папки перейдите на вкладку **Доступ**
   - Нажмите **Расширенная настройка общего доступа** (Advanced Sharing)
   - Поставьте галочку **Открыть общий доступ к этой папке** (Share this folder)
   - В поле **Имя общего ресурса** (Share name) оставьте `Network`
   - Нажмите **Разрешения** (Permissions)
   - Добавьте пользователя `Everyone` и дайте **Полный доступ**
   - Нажмите ОК дважды

2.4. **Проверьте доступность сетевой папки через xp_cmdshell:**


2.5. **Узнайте имя вашего компьютера:**

```sql

-- или через xp_cmdshell

GO
```

Запишите имя компьютера, например: `DESKTOP-ABC123`

2.6. **Проверьте доступ по имени компьютера:**

```sql

```

2.7. **Проверьте доступ через Проводник:**
   - Нажмите Win + R, введите `\\localhost\Network` и нажмите Enter
   - Должна открыться пустая папка

### 🔹 Этап 3. Проблемы с правами доступа (15 мин)

**Важно:** SQL Server работает под определённой учётной записью. Ей нужен доступ к сетевой папке, даже если она на том же компьютере!

3.1. **Узнайте, под какой учётной записью работает SQL Server:**

```sql
-- Учётная запись службы SQL Server
SELECT servicename, service_account
FROM sys.dm_server_services;
GO

-- Альтернативный способ
EXEC xp_cmdshell 'whoami';
GO
```

Обычно это:
- `NT SERVICE\MSSQLSERVER` (для экземпляра по умолчанию)
- `NT SERVICE\MSSQL$SQLEXPRESS` (для Express)
- `NT AUTHORITY\NETWORK SERVICE`
- Или доменная учётная запись

**Запишите эту учётную запись!**

3.2. **Попробуйте выполнить бэкап в сетевую папку (должна быть ошибка):**

```sql
BACKUP DATABASE NetworkBackup_Фамилия
TO DISK = N'\\localhost\Network\NetworkBackup_Фамилия.bak'
WITH COMPRESSION, CHECKSUM;
GO
```

**Запишите текст ошибки.** Обычно это "Operating system error 5(Access is denied)".

3.3. **Предоставьте права доступа через Проводник:**

   - Откройте Проводник и найдите папку `C:\Backup\ПР22\Network`
   - Правой кнопкой мыши → **Свойства**
   - Перейдите на вкладку **Безопасность** (Security)
   - Нажмите кнопку **Изменить...** (Edit)
   - Нажмите **Добавить...** (Add)
   - В поле "Введите имена объектов" введите учётную запись SQL Server, которую вы определили в п.3.1
     Например: `NT SERVICE\MSSQLSERVER`
   - Нажмите **Проверить имена** (Check Names) — если имя подчеркнётся, значит всё правильно
   - Нажмите **ОК**
   - В списке разрешений дайте **Полный доступ** (Full Control)
   - Нажмите **ОК**

3.4. **Если вы не можете найти учётную запись SQL Server**, можно дать доступ группе `Everyone` (для учебных целей):

   - Добавьте пользователя `Everyone`
   - Дайте полный доступ
   - Нажмите ОК

3.5. **Предоставьте права через командную строку (альтернативный способ):**

```sql
-- Дать права группе Everyone (выполнить от имени администратора)
EXEC xp_cmdshell 'icacls C:\Backup\ПР22\Network /grant "Everyone:(OI)(CI)F"';
GO

-- Или для конкретной учётной записи SQL Server
EXEC xp_cmdshell 'icacls C:\Backup\ПР22\Network /grant "NT SERVICE\MSSQLSERVER:(OI)(CI)F"';
GO
```

3.6. **Повторите бэкап** — теперь он должен работать:

```sql
BACKUP DATABASE NetworkBackup_Фамилия
TO DISK = N'\\localhost\Network\NetworkBackup_Фамилия.bak'
WITH COMPRESSION, CHECKSUM;
GO
```

3.7. **Проверьте, что файл создался через xp_cmdshell:**

```sql
EXEC xp_cmdshell 'dir \\localhost\Network\*.bak';
GO
```

3.8. **Проверьте через Проводник:**
   - Откройте `\\localhost\Network`
   - Вы должны увидеть файл `NetworkBackup_Фамилия.bak`

3.9. **Сделайте скриншоты** ошибки, окна настроек безопасности и успешного выполнения.

### 🔹 Этап 5. Создание хранимой процедуры для бэкапа в сеть (20 мин)

5.1. **Создайте процедуру для бэкапа в сетевую папку с проверками:**

```sql
USE master;
GO

CREATE OR ALTER PROCEDURE dbo.usp_NetworkBackup
    @DatabaseName NVARCHAR(128),
    @NetworkPath NVARCHAR(255) = '\\localhost\Network\',
    @CompressionEnabled BIT = 1,
    @ChecksumEnabled BIT = 1
AS
BEGIN
    SET NOCOUNT ON;
    
    DECLARE @FileName NVARCHAR(500);
    DECLARE @DateStr NVARCHAR(20) = FORMAT(GETDATE(), 'yyyyMMdd_HHmm');
    DECLARE @SQL NVARCHAR(MAX);
    DECLARE @StartTime DATETIME = GETDATE();
    DECLARE @TestFile NVARCHAR(500);
    
    -- Проверяем, существует ли база
    IF NOT EXISTS (SELECT 1 FROM sys.databases WHERE name = @DatabaseName)
    BEGIN
        RAISERROR('База данных %s не существует', 16, 1, @DatabaseName);
        RETURN;
    END
    
    -- Проверяем доступность сетевой папки
    SET @TestFile = @NetworkPath + 'test_' + @DateStr + '.txt';
    
    BEGIN TRY
        DECLARE @TestSQL NVARCHAR(1000) = 
            'EXEC xp_cmdshell ''echo Test > ' + @TestFile + '''';
        EXEC sp_executesql @TestSQL;
        
        -- Удаляем тестовый файл
        DECLARE @DelSQL NVARCHAR(1000) = 
            'EXEC xp_cmdshell ''del ' + @TestFile + '''';
        EXEC sp_executesql @DelSQL;
        
        PRINT 'Сетевая папка доступна.';
    END TRY
    BEGIN CATCH
        RAISERROR('Сетевая папка %s недоступна. Проверьте права доступа.', 
                  16, 1, @NetworkPath);
        RETURN;
    END CATCH
    
    -- Генерируем имя файла
    SET @FileName = @NetworkPath + @DatabaseName + '_' + @DateStr + '.bak';
    
    -- Формируем команду бэкапа
    SET @SQL = 'BACKUP DATABASE [' + @DatabaseName + '] TO DISK = ''' + @FileName + '''';
    
    IF @CompressionEnabled = 1
        SET @SQL = @SQL + ' WITH COMPRESSION';
    ELSE
        SET @SQL = @SQL + ' WITH NO_COMPRESSION';
    
    IF @ChecksumEnabled = 1
        SET @SQL = @SQL + ', CHECKSUM';
    
    SET @SQL = @SQL + ', STATS = 10;';
    
    -- Выводим информацию
    PRINT 'Начало бэкапа: ' + @DatabaseName;
    PRINT 'Файл: ' + @FileName;
    PRINT 'Время: ' + CONVERT(NVARCHAR, @StartTime, 120);
    
    -- Выполняем бэкап
    BEGIN TRY
        EXEC sp_executesql @SQL;
        
        DECLARE @EndTime DATETIME = GETDATE();
        DECLARE @Duration INT = DATEDIFF(second, @StartTime, @EndTime);
        
        PRINT 'Бэкап успешно завершён за ' + CAST(@Duration AS NVARCHAR) + ' сек.';
        
        -- Возвращаем информацию о файле
        SELECT @FileName AS BackupFileName, @Duration AS DurationSeconds;
    END TRY
    BEGIN CATCH
        PRINT 'ОШИБКА при бэкапе: ' + ERROR_MESSAGE();
        THROW;
    END CATCH
END;
GO
```

5.2. **Протестируйте процедуру:**

```sql

    @NetworkPath = '\\localhost\Network\',
```

### 🔹 Этап 6. Сравнение локального и сетевого хранения (10 мин)

6.1. **Создайте тест для сравнения скорости:**

```sql
-- Засекаем время локального бэкапа (в папку C:\Backup\ПР22\Local)
DECLARE @StartTime DATETIME = GETDATE();

BACKUP DATABASE NetworkBackup_Фамилия
TO DISK = 'C:\Backup\ПР22\Local\LocalBackup.bak'
WITH COMPRESSION, CHECKSUM;

DECLARE @LocalDuration INT = DATEDIFF(second, @StartTime, GETDATE());
PRINT 'Локальный бэкап: ' + CAST(@LocalDuration AS NVARCHAR) + ' сек.';

-- Засекаем время сетевого бэкапа (в \\localhost\Network)
SET @StartTime = GETDATE();

BACKUP DATABASE NetworkBackup_Фамилия
TO DISK = '\\localhost\Network\NetworkBackup.bak'
WITH COMPRESSION, CHECKSUM;

DECLARE @NetworkDuration INT = DATEDIFF(second, @StartTime, GETDATE());
PRINT 'Сетевой бэкап: ' + CAST(@NetworkDuration AS NVARCHAR) + ' сек.';

-- Сравнение
IF @LocalDuration < @NetworkDuration
    PRINT 'Локальный бэкап быстрее на ' + CAST(@NetworkDuration - @LocalDuration AS NVARCHAR) + ' сек.';
ELSE IF @LocalDuration > @NetworkDuration
    PRINT 'Сетевой бэкап быстрее на ' + CAST(@LocalDuration - @NetworkDuration AS NVARCHAR) + ' сек.';
ELSE
    PRINT 'Скорость одинаковая';
GO
```

6.2. **Проанализируйте результаты.** Обычно локальный бэкап немного быстрее, но разница может быть незначительной, так как диск один и тот же.

6.3. **Проверьте созданные файлы:**

```sql
-- Проверка локальных файлов
EXEC xp_cmdshell 'dir C:\Backup\ПР22\Local\*.bak';
GO

-- Проверка сетевых файлов
EXEC xp_cmdshell 'dir \\localhost\Network\*.bak';
GO
```

6.4. **Проверьте через Проводник:**
   - Откройте `C:\Backup\ПР22\Local` — там должен быть файл `LocalBackup.bak`
   - Откройте `\\localhost\Network` — там должен быть файл `NetworkBackup.bak`

## 📌 Полезные команды для справки

### Управление xp_cmdshell:
```sql
-- Включение
EXEC sp_configure 'show advanced options', 1;
RECONFIGURE;
EXEC sp_configure 'xp_cmdshell', 1;
RECONFIGURE;

-- Отключение
EXEC sp_configure 'xp_cmdshell', 0;
RECONFIGURE;
EXEC sp_configure 'show advanced options', 0;
RECONFIGURE;

-- Проверка состояния
EXEC sp_configure 'xp_cmdshell';
```

### Полезные команды xp_cmdshell:
```sql
-- Создание папки
EXEC xp_cmdshell 'mkdir C:\Путь\К\Папке';

-- Просмотр содержимого папки
EXEC xp_cmdshell 'dir C:\Путь\К\Папке';

-- Копирование файла
EXEC xp_cmdshell 'copy C:\file1.txt D:\file2.txt';

-- Удаление файла
EXEC xp_cmdshell 'del C:\file.txt';

-- Проверка доступности сетевой папки
EXEC xp_cmdshell 'dir \\localhost\Network';
```

### Настройка прав через командную строку:
```cmd
icacls C:\Backup\ПР22\Network /grant "Everyone:(OI)(CI)F"
icacls C:\Backup\ПР22\Network /grant "NT SERVICE\MSSQLSERVER:(OI)(CI)F"
```

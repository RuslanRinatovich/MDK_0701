# 🧪 Практическая работа №18  
## Восстановление из полной резервной копии

⏱️ **Продолжительность:** 90 минут  
🎯 **Цель работы:**  
Научиться восстанавливать базы данных из полных резервных копий в различных сценариях: восстановление поверх существующей базы, восстановление с перемещением файлов, восстановление на другой сервер. Освоить работу с состояниями RECOVERY и NORECOVERY, научиться диагностировать ошибки восстановления.

🖥️ **Оборудование:**  
Виртуальная машина с Windows Server / Windows 10, установленными SQL Server Developer Edition и SSMS.

## 🧠 Глубокое погружение: перед началом

- Вспомните, чем отличается `WITH RECOVERY` от `WITH NORECOVERY`.
- Подумайте, в каких случаях может потребоваться восстановление с перемещением файлов.
- Что произойдёт, если попытаться восстановить полную копию в существующую базу без `REPLACE`?

## 📋 Ход выполнения работы

### 🔹 Этап 1. Подготовка среды и создание двух версий бэкапов (20 мин)

1. Запустите **SSMS** и подключитесь к экземпляру SQL Server.

2. Создайте исходную базу данных с уникальным именем:

```sql
CREATE DATABASE RestorePractice_Фамилия;
GO
USE RestorePractice_Фамилия;
GO
```

3. Создайте таблицы и наполните их начальными данными:

```sql
-- Таблица сотрудников
CREATE TABLE dbo.Employees (
    EmployeeID INT IDENTITY(1,1) PRIMARY KEY,
    LastName NVARCHAR(50),
    FirstName NVARCHAR(50),
    Salary DECIMAL(10,2),
    Department NVARCHAR(50)
);

-- Таблица заказов
CREATE TABLE dbo.Orders (
    OrderID INT IDENTITY(1,1) PRIMARY KEY,
    EmployeeID INT REFERENCES dbo.Employees(EmployeeID),
    OrderDate DATETIME DEFAULT GETDATE(),
    Amount DECIMAL(10,2),
    Status NVARCHAR(20)
);

-- Наполнение начальными данными
INSERT INTO dbo.Employees (LastName, FirstName, Salary, Department)
VALUES 
    ('Иванов', 'Иван', 75000, 'IT'),
    ('Петров', 'Петр', 82000, 'IT'),
    ('Сидорова', 'Анна', 95000, 'Finance'),
    ('Козлов', 'Дмитрий', 67000, 'Sales');

INSERT INTO dbo.Orders (EmployeeID, Amount, Status)
VALUES 
    (1, 1500.00, 'Выполнен'),
    (2, 2300.50, 'В обработке'),
    (1, 800.75, 'Новый'),
    (3, 3200.00, 'Выполнен');
GO
```

4. Создайте папку `C:\Backup\ПР18\` для хранения бэкапов.

5. **Создание первой полной копии (версия 1 – без Новикова):**

```sql
BACKUP DATABASE [RestorePractice_Фамилия]
TO DISK = N'C:\Backup\ПР18\RestorePractice_Фамилия_Full_V1.bak'
WITH INIT, COMPRESSION, CHECKSUM, STATS = 10;
```

6. **Добавьте новые данные (сотрудник Новиков и его заказ):**

```sql
INSERT INTO dbo.Employees (LastName, FirstName, Salary, Department)
VALUES ('Новиков', 'Андрей', 72000, 'IT');

INSERT INTO dbo.Orders (EmployeeID, Amount, Status)
VALUES (5, 1750.00, 'Новый');
GO
```

7. **Создание второй полной копии (версия 2 – с Новиковым):**

```sql
BACKUP DATABASE [RestorePractice_Фамилия]
TO DISK = N'C:\Backup\ПР18\RestorePractice_Фамилия_Full_V2.bak'
WITH INIT, COMPRESSION, CHECKSUM, STATS = 10;
```

8. **Сделайте скриншоты** обеих команд бэкапа и результатов выполнения. Убедитесь, что на скриншотах видны команды и сообщения об успешном завершении.

### 🔹 Этап 2. Простое восстановление первой версии (15 мин)

1. **Удалите исходную базу** (имитация катастрофы):

```sql
USE master;
GO
DROP DATABASE RestorePractice_Фамилия;
GO
```

2. **Восстановите первую версию базы (без Новикова):**

```sql
RESTORE DATABASE RestorePractice_Фамилия
FROM DISK = N'C:\Backup\ПР18\RestorePractice_Фамилия_Full_V1.bak'
WITH RECOVERY;
```

3. **Проверьте, что база восстановлена и содержит данные:**

```sql
USE RestorePractice_Фамилия;
GO
SELECT * FROM dbo.Employees;
SELECT * FROM dbo.Orders;
```

4. **Вопрос:** Есть ли в восстановленной базе запись о сотруднике "Новиков"? Почему?

5. **Сделайте скриншоты** команд и результатов запросов.

### 🔹 Этап 4. Восстановление с перемещением файлов (MOVE) (15 мин)

1. **Удалите базу (если ещё не удалена):**

```sql
USE master;
GO
DROP DATABASE IF EXISTS RestorePractice_Фамилия;
GO
```

2. **Сначала узнайте логические имена файлов в бэкапе:**

```sql
RESTORE FILELISTONLY 
FROM DISK = N'C:\Backup\ПР18\RestorePractice_Фамилия_Full_V2.bak';
```

3. Запишите логические имена файлов данных и журнала (обычно это имя базы и имя_базы_log).

4. Создайте папку `C:\Restored\` (если её нет).

5. **Выполните восстановление с перемещением файлов в новую папку:**

```sql
RESTORE DATABASE RestorePractice_Фамилия_Moved
FROM DISK = N'C:\Backup\ПР18\RestorePractice_Фамилия_Full_V2.bak'
WITH 
    MOVE 'RestorePractice_Фамилия' TO 'C:\Restored\RestorePractice_Фамилия.mdf',
    MOVE 'RestorePractice_Фамилия_log' TO 'C:\Restored\RestorePractice_Фамилия_log.ldf',
    RECOVERY;
```

6. **Проверьте, что база `RestorePractice_Фамилия_Moved` существует и содержит данные (включая Новикова).**

7. **Сделайте скриншоты** всех шагов: `RESTORE FILELISTONLY`, команды восстановления с MOVE и проверки данных.

### 🔹 Этап 5. Восстановление с NORECOVERY и разностной копией (20 мин)

1. **Убедитесь, что база `RestorePractice_Фамилия` существует и содержит Новикова.**  
   Если база была удалена, восстановите её из второй полной копии:

```sql
RESTORE DATABASE RestorePractice_Фамилия
FROM DISK = N'C:\Backup\ПР18\RestorePractice_Фамилия_Full_V2.bak'
WITH RECOVERY;
```

2. **Добавьте нового сотрудника (Смирнова) для создания изменений:**

```sql
USE RestorePractice_Фамилия;
GO
INSERT INTO dbo.Employees (LastName, FirstName, Salary, Department)
VALUES ('Смирнова', 'Елена', 89000, 'Finance');
GO
```

3. **Создайте разностную резервную копию** (фиксируем изменения):

```sql
BACKUP DATABASE [RestorePractice_Фамилия]
TO DISK = N'C:\Backup\ПР18\RestorePractice_Фамилия_Diff.bak'
WITH DIFFERENTIAL, COMPRESSION, CHECKSUM;
```

4. **Удалите базу** (имитация новой катастрофы):

```sql
USE master;
GO
DROP DATABASE RestorePractice_Фамилия;
GO
```

5. **Восстановите полную копию V2 с NORECOVERY:**

```sql
RESTORE DATABASE RestorePractice_Фамилия
FROM DISK = N'C:\Backup\ПР18\RestorePractice_Фамилия_Full_V2.bak'
WITH NORECOVERY;
```

6. **Проверьте состояние базы** – она должна быть в режиме `Restoring` (недоступна для пользователей). Попробуйте выполнить:

```sql
USE RestorePractice_Фамилия; -- Должна быть ошибка
```

7. **Примените разностную копию:**

```sql
RESTORE DATABASE RestorePractice_Фамилия
FROM DISK = N'C:\Backup\ПР18\RestorePractice_Фамилия_Diff.bak'
WITH RECOVERY;
```

8. **Проверьте наличие Смирновой** (она должна появиться после применения разностной копии):

```sql
USE RestorePractice_Фамилия;
GO
SELECT * FROM dbo.Employees WHERE LastName = 'Смирнова';
```

9. **Сделайте скриншоты** всех ключевых шагов.

### 🔹 Этап 6. Моделирование ошибок и их диагностика (15 мин)

1. **Попробуйте восстановить разностную копию без полной:**

-- Попытка восстановить только разностную копию

2. **Запишите текст ошибки.** Понимаете ли вы, почему она возникла?

3. **Попробуйте восстановить базу поверх существующей без REPLACE:**

```sql
-- Сначала восстановим базу из полной копии
RESTORE DATABASE RestorePractice_Фамилия
FROM DISK = N'C:\Backup\ПР18\RestorePractice_Фамилия_Full_V2.bak'
WITH RECOVERY;

-- Теперь попробуем восстановить поверх неё без REPLACE
RESTORE DATABASE RestorePractice_Фамилия

WITH RECOVERY;
```

4. **Запишите текст ошибки.**

5. **Исправьте ситуацию с помощью `WITH REPLACE`:**

```sql
RESTORE DATABASE RestorePractice_Фамилия
FROM DISK = N'C:\Backup\ПР18\RestorePractice_Фамилия_Full_V2.bak'
WITH REPLACE, RECOVERY;
```

6. **Сделайте скриншоты** всех ошибок и успешного восстановления с REPLACE.

### 🔹 Этап 7. Анализ истории восстановлений (10 мин)

1. **Просмотрите историю восстановлений на сервере:**

```sql
SELECT 
    destination_database_name,
    restore_date,
    restore_type,
    user_name,
    CASE replace WHEN 1 THEN 'Да' ELSE 'Нет' END AS Было_REPLACE,
    CASE recovery WHEN 1 THEN 'Да' ELSE 'Нет' END AS Было_RECOVERY
FROM msdb.dbo.restorehistory
WHERE destination_database_name LIKE '%RestorePractice%'
ORDER BY restore_date DESC;
```

2. **Проверьте, какие файлы были созданы при восстановлении с MOVE:**

```sql
-- Выполните в командной строке или через xp_cmdshell
EXEC xp_cmdshell 'dir C:\Restored\';
```

7. Что означает состояние базы "Restoring"?

9. Почему после восстановления важно сделать новый полный бэкап?
10. Где SQL Server хранит историю восстановлений?
11. Чем отличаются две версии полных копий, созданные в этапе 1?
12. Как убедиться, что восстановление с NORECOVERY прошло успешно, до применения следующих копий?

## 📊 Краткие критерии оценивания

| Критерий | «5» (отлично) | «4» (хорошо) | «3» (удовл.) |
|----------|----------------|---------------|---------------|
| **Стартовый скрипт и две версии бэкапов** | БД создана, обе версии бэкапов, скриншоты | БД создана, но одна версия | Ошибки или не выполнено |
| **Этап 2 (восстановление V1)** | Успешно, проверка, понимание отсутствия Новикова | Выполнено, но нет проверки | Не выполнено |
| **Этап 3 (восстановление V2)** | Успешно, проверка наличия Новикова, скриншот | Выполнено, но нет проверки | Не выполнено |
| **Этап 4 (восстановление с MOVE)** | FILELISTONLY, MOVE, проверка, скриншоты | Выполнено, но нет FILELISTONLY | Не выполнено |
| **Этап 5 (NORECOVERY + DIFF)** | Полная цепочка, добавление Смирновой, проверка | Частично выполнено | Не выполнено |
| **Этап 6 (ошибки)** | Смоделированы все 3 ошибки, анализ | Часть ошибок пропущена | Не выполнено |

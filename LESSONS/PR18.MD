# 🧪 Практическая работа №18  
## Восстановление из полной резервной копии

⏱️ **Продолжительность:** 90 минут  
🎯 **Цель работы:**  
Научиться восстанавливать базы данных из полных резервных копий в различных сценариях: восстановление поверх существующей базы, восстановление с перемещением файлов, восстановление на другой сервер. Освоить работу с состояниями RECOVERY и NORECOVERY, научиться диагностировать ошибки восстановления.

🖥️ **Оборудование:**  
Виртуальная машина с Windows Server / Windows 10, установленными SQL Server Developer Edition и SSMS.

---

## 📋 Общая информация  

| Параметр | Значение |
|----------|----------|
| Время выполнения | 90 минут |
| Количество заданий | 6 |

## 🧠 Глубокое погружение: перед началом

- Вспомните, чем отличается `WITH RECOVERY` от `WITH NORECOVERY`.
- Подумайте, в каких случаях может потребоваться восстановление с перемещением файлов.
- Что произойдёт, если попытаться восстановить полную копию в существующую базу без `REPLACE`?

## 📋 Ход выполнения работы

### 🔹 Этап 1. Подготовка среды (15 мин)

1. Запустите **SSMS** и подключитесь к экземпляру SQL Server.

2. Создайте исходную базу данных с уникальным именем:

```sql
CREATE DATABASE RestorePractice_Фамилия;
GO
USE RestorePractice_Фамилия;
GO
```

3. Создайте таблицы и наполните их данными:

```sql
-- Таблица сотрудников
CREATE TABLE dbo.Employees (
    EmployeeID INT IDENTITY(1,1) PRIMARY KEY,
    LastName NVARCHAR(50),
    FirstName NVARCHAR(50),
    Salary DECIMAL(10,2),
    Department NVARCHAR(50)
);

-- Таблица заказов
CREATE TABLE dbo.Orders (
    OrderID INT IDENTITY(1,1) PRIMARY KEY,
    EmployeeID INT REFERENCES dbo.Employees(EmployeeID),
    OrderDate DATETIME DEFAULT GETDATE(),
    Amount DECIMAL(10,2),
    Status NVARCHAR(20)
);

-- Наполнение данными
INSERT INTO dbo.Employees (LastName, FirstName, Salary, Department)
VALUES 
    ('Иванов', 'Иван', 75000, 'IT'),
    ('Петров', 'Петр', 82000, 'IT'),
    ('Сидорова', 'Анна', 95000, 'Finance'),
    ('Козлов', 'Дмитрий', 67000, 'Sales');

INSERT INTO dbo.Orders (EmployeeID, Amount, Status)
VALUES 
    (1, 1500.00, 'Выполнен'),
    (2, 2300.50, 'В обработке'),
    (1, 800.75, 'Новый'),
    (3, 3200.00, 'Выполнен');
GO
```

4. Создайте папку `C:\Backup\ПР18\` для хранения бэкапов.

5. Создайте полную резервную копию:

```sql
BACKUP DATABASE [RestorePractice_Фамилия]
TO DISK = N'C:\Backup\ПР18\RestorePractice_Фамилия_Full.bak'
WITH INIT, COMPRESSION, CHECKSUM, STATS = 10;
```

6. **Скриншот** выполнения бэкапа.

7. После создания бэкапа **добавьте новые данные** (имитация работы после бэкапа):

```sql
INSERT INTO dbo.Employees (LastName, FirstName, Salary, Department)
VALUES ('Новиков', 'Андрей', 72000, 'IT');

INSERT INTO dbo.Orders (EmployeeID, Amount, Status)
VALUES (5, 1750.00, 'Новый');
GO
```

### 🔹 Этап 2. Простое восстановление с REPLACE (15 мин)

1. **Удалите исходную базу** (имитация катастрофы):

```sql
USE master;
GO
DROP DATABASE RestorePractice_Фамилия;
GO
```

2. **Восстановите базу из полной копии:**

```sql
RESTORE DATABASE RestorePractice_Фамилия
FROM DISK = N'C:\Backup\ПР18\RestorePractice_Фамилия_Full.bak'
WITH RECOVERY;
```

3. **Проверьте, что база восстановлена и содержит данные:**

```sql
USE RestorePractice_Фамилия;
GO
SELECT * FROM dbo.Employees;
SELECT * FROM dbo.Orders;
```

4. **Вопрос:** Есть ли в восстановленной базе запись о сотруднике "Новиков"? Почему?

5. **Скриншоты** команд и результатов.

---

### 🔹 Этап 3. Восстановление с перемещением файлов (MOVE) (15 мин)

1. **Снова удалите базу** (или создайте новую для теста, но проще удалить и восстановить по-новой).

2. **Восстановите базу, но переместите файлы в другую папку:**

```sql
-- Сначала узнайте логические имена файлов в бэкапе
RESTORE FILELISTONLY 
FROM DISK = N'C:\Backup\ПР18\RestorePractice_Фамилия_Full.bak';
```

3. Запишите логические имена файлов данных и журнала (обычно как имя базы и имя_базы_log).

4. Создайте папку `C:\Restored\` (если нет).

5. Выполните восстановление с перемещением:

```sql
RESTORE DATABASE RestorePractice_Фамилия_Moved
FROM DISK = N'C:\Backup\ПР18\RestorePractice_Фамилия_Full.bak'
WITH 
    MOVE 'RestorePractice_Фамилия' TO 'C:\Restored\RestorePractice_Фамилия.mdf',
    MOVE 'RestorePractice_Фамилия_log' TO 'C:\Restored\RestorePractice_Фамилия_log.ldf',
    RECOVERY;
```

6. **Проверьте, что база `RestorePractice_Фамилия_Moved` существует и содержит данные.**

7. **Скриншоты** всех шагов.

---

### 🔹 Этап 4. Восстановление с NORECOVERY и последующим применением изменений (20 мин)

1. **Создайте новую полную копию текущего состояния** (с данными, включая "Новикова"):

```sql
BACKUP DATABASE [RestorePractice_Фамилия]
TO DISK = N'C:\Backup\ПР18\RestorePractice_Фамилия_Full2.bak'
WITH INIT, COMPRESSION, CHECKSUM;
```

2. **Создайте разностную копию** (для усложнения):

```sql
-- Добавим ещё данные
INSERT INTO dbo.Employees (LastName, FirstName, Salary, Department)
VALUES ('Смирнова', 'Елена', 89000, 'Finance');
GO

-- Разностная копия
BACKUP DATABASE [RestorePractice_Фамилия]
TO DISK = N'C:\Backup\ПР18\RestorePractice_Фамилия_Diff.bak'
WITH DIFFERENTIAL, COMPRESSION, CHECKSUM;
```

3. **Удалите базу.**

4. **Восстановите полную копию с NORECOVERY:**

```sql
RESTORE DATABASE RestorePractice_Фамилия
FROM DISK = N'C:\Backup\ПР18\RestorePractice_Фамилия_Full2.bak'
WITH NORECOVERY;
```

5. **Проверьте состояние базы** – она должна быть в режиме `Restoring` (недоступна).

6. **Примените разностную копию:**

```sql
RESTORE DATABASE RestorePractice_Фамилия
FROM DISK = N'C:\Backup\ПР18\RestorePractice_Фамилия_Diff.bak'
WITH RECOVERY;  -- сразу завершаем
```

7. **Проверьте наличие данных** – должен быть сотрудник "Смирнова".

8. **Скриншоты** всех шагов.

---

### 🔹 Этап 5. Моделирование ошибок и их диагностика (15 мин)

1. **Попробуйте восстановить разностную копию без полной:**

```sql
-- Удалим базу
DROP DATABASE IF EXISTS RestorePractice_Фамилия;

-- Попытка восстановить только разностную
RESTORE DATABASE RestorePractice_Фамилия
FROM DISK = N'C:\Backup\ПР18\RestorePractice_Фамилия_Diff.bak'
WITH RECOVERY;
```

2. **Запишите текст ошибки.** Понимаете ли вы, почему она возникла?

3. **Попробуйте восстановить базу поверх существующей без REPLACE:**

```sql
-- База уже существует, пробуем восстановить
RESTORE DATABASE RestorePractice_Фамилия
FROM DISK = N'C:\Backup\ПР18\RestorePractice_Фамилия_Full2.bak'
WITH RECOVERY;
```

4. **Запишите текст ошибки.**

5. **Исправьте ситуацию с помощью `WITH REPLACE`:**

```sql
RESTORE DATABASE RestorePractice_Фамилия
FROM DISK = N'C:\Backup\ПР18\RestorePractice_Фамилия_Full2.bak'
WITH REPLACE, RECOVERY;
```

6. **Скриншоты** всех ошибок и успешного восстановления.

### 🔹 Этап 6. Анализ восстановления (10 мин)

1. **Просмотрите историю восстановлений на сервере:**

```sql
SELECT 
    destination_database_name,
    restore_date,
    restore_type,
    user_name,
    replace,
    recovery
FROM msdb.dbo.restorehistory
WHERE destination_database_name LIKE '%RestorePractice%'
ORDER BY restore_date DESC;
```

2. **Проверьте, какие файлы были созданы при восстановлении с MOVE:**

```sql
EXEC xp_cmdshell 'dir C:\Restored\';
```

3. **Скриншоты** результатов.

## 📊 Краткие критерии оценивания

| Критерий | «5» (отлично) | «4» (хорошо) | «3» (удовл.) |
|----------|----------------|---------------|---------------|
| **Стартовый скрипт и бэкап** | БД с фамилией, таблицы, данные, бэкап | БД создана, но мелкие недочёты | Ошибки или не выполнен |
| **Этап 2 (простое восстановление)** | Успешно, проверка данных, скриншот | Выполнено, но нет проверки | Не выполнено |
| **Этап 3 (восстановление с MOVE)** | Правильно определены имена, перемещение, скриншот | Выполнено, но нет FILELISTONLY | Не выполнено |
| **Этап 4 (NORECOVERY + DIFF)** | Полная + разностная с NORECOVERY, проверка | Частично выполнено | Не выполнено |
| **Этап 5 (ошибки)** | Смоделированы все ошибки, анализ | Часть ошибок пропущена | Не выполнено |
| **Этап 6 (анализ)** | Запрос к restorehistory, анализ | Запрос есть, анализ поверхностный | Не выполнено |
| **Таблица наблюдений** | Полностью заполнена | 1-2 пропуска | Отсутствует |
| **Ответы на вопросы** | Полные, с терминами | 2-3 неполных | Большинства нет |
| **Оформление отчёта** | Структура, подписи, вывод | Мелкие нарушения | Неструктурирован |

**Штрафы:** -1 за отсутствие скриншотов, -0.5 за нечитаемые, -1 за отсутствие вывода  
❗ **Без таблицы — оценка не выше «4»**

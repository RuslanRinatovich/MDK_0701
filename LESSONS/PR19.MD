# 🧪 Практическая работа №19  
## Моделирование ошибки → восстановление до момента до неё

⏱️ **Продолжительность:** 90 минут  
🎯 **Цель работы:**  
Смоделировать реальную аварийную ситуацию (случайное удаление данных), выполнить восстановление базы данных на момент времени непосредственно перед ошибкой, используя механизм Point-in-Time Recovery (PITR) с параметром STOPAT. Научиться определять точное время сбоя, работать с цепочкой резервных копий журнала транзакций и восстанавливать базу с транзакционной согласованностью.

🖥️ **Оборудование:**  
Виртуальная машина с Windows Server / Windows 10, установленными SQL Server Developer Edition и SSMS.

## 🧠 Глубокое погружение: перед началом

Вспомните ключевые требования для PITR :
- Модель восстановления **FULL** (только она поддерживает восстановление на момент времени)
- Наличие **полной резервной копии** (отправная точка)
- **Непрерывная цепочка журналов** транзакций от полной копии до целевого момента
- Знание **точного времени** события (или использование маркеров транзакций)

> ⚠️ **Важно:** Если в журнале есть операции с минимальным логированием (BULK_LOGGED), восстановление на момент времени может быть невозможным .

## 📋 Ход выполнения работы

### 🔹 Этап 1. Подготовка среды (15 мин)

1. Запустите **SSMS** и подключитесь к экземпляру SQL Server.

2. Создайте базу данных с уникальным именем:

```sql
CREATE DATABASE PITRPractice_Фамилия;
GO
USE PITRPractice_Фамилия;
GO
```

3. Убедитесь, что модель восстановления **FULL** (это критически важно для PITR):

```sql
SELECT name, recovery_model_desc 
FROM sys.databases 
WHERE name = DB_NAME();

-- Если не FULL:
ALTER DATABASE PITRPractice_Фамилия SET RECOVERY FULL;
GO
```

4. Создайте таблицы для экспериментов:

```sql
-- Таблица продуктов
CREATE TABLE dbo.Products (
    ProductID INT IDENTITY(1,1) PRIMARY KEY,
    ProductName NVARCHAR(100),
    Price DECIMAL(10,2),
    Quantity INT,
    Category NVARCHAR(50)
);

-- Таблица заказов
CREATE TABLE dbo.Orders (
    OrderID INT IDENTITY(1,1) PRIMARY KEY,
    ProductID INT,
    OrderDate DATETIME DEFAULT GETDATE(),
    Quantity INT,
    TotalAmount DECIMAL(10,2),
    Status NVARCHAR(20)
);

-- Таблица для лога операций
CREATE TABLE dbo.OperationLog (
    LogID INT IDENTITY(1,1) PRIMARY KEY,
    OperationTime DATETIME DEFAULT GETDATE(),
    OperationName NVARCHAR(100),
    Details NVARCHAR(500)
);
GO
```

5. Создайте папку для хранения бэкапов: `C:\Backup\ПР19\`

6. Наполните таблицы начальными данными :

```sql

    (4, 1, 25000, 'Новый');
```

### 🔹 Этап 4. Анализ ситуации и подготовка к восстановлению (15 мин)

4.1. **Проверьте, какие резервные копии у нас есть:**

```sql
SELECT 
    database_name,
    backup_start_date,
    backup_finish_date,
    CASE type
        WHEN 'D' THEN 'Полная'
        WHEN 'L' THEN 'Журнал'
    END AS BackupType,
    first_lsn,
    last_lsn,
    backup_size/1048576 AS SizeMB,
    physical_device_name
FROM msdb.dbo.backupset bs
JOIN msdb.dbo.backupmediafamily bmf 
    ON bs.media_set_id = bmf.media_set_id
WHERE bs.database_name = 'PITRPractice_Фамилия'
ORDER BY backup_start_date;
```

4.2. **Определите целевое время для восстановления** :

Целевое время должно быть **непосредственно перед** выполнением DELETE, но после всех предыдущих операций. Используйте время, которое вы записали в п. 3.1, и **вычтите 1-2 секунды** (чтобы точно оказаться до удаления).

Например, если DisasterTime = '2026-03-10 10:30:00.000', то целевое время = '2026-03-10 10:29:58.000'

4.3. **Проверьте, можно ли восстановиться на этот момент** :

```sql
-- Проверка наличия журналов до нужного момента
SELECT 
    backup_start_date,
    backup_finish_date,
    first_lsn,
    last_lsn
FROM msdb.dbo.backupset
WHERE database_name = 'PITRPractice_Фамилия'
    AND type = 'L'
    AND backup_start_date < '2026-03-10 10:30:00'  -- укажите ваше время
ORDER BY backup_start_date;
```
